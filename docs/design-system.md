# Design System

## Philosophy

The Atlas9 design system embodies a **High-End Research Lab** aesthetic. Every element communicates precision, scientific rigor, and elite craftsmanship. The system is designed to make generated products look better than Vercel, Linear, Stripe, Notion, and Figma.

Design is infrastructure, not decoration.

## Core Principles

1. **Precision Over Decoration** — Every pixel has purpose
2. **Restraint Over Excess** — Show confidence through what you don't do
3. **Density Over Sprawl** — Information-rich without clutter
4. **Motion Over Static** — Animation communicates state and causality
5. **Accessibility Over Exclusivity** — Beautiful for everyone

## Typography

### Type Families

| Role | Family | Fallback | Usage |
|------|--------|----------|-------|
| Serif | `var(--font-serif)` | Georgia, serif | Headings, hero text, narrative, brand moments |
| Mono | `var(--font-mono)` | ui-monospace, monospace | Code, data, timestamps, IDs, technical labels |
| Sans | `var(--font-sans)` | system-ui, sans-serif | UI labels, buttons, navigation, body |

### Type Scale

```css
--text-xs: 0.75rem;     /* 12px - Captions, labels */

--text-lg: 1.125rem;    /* 18px - Large body */
--text-xl: 1.25rem;     /* 20px - Small headings */
--text-2xl: 1.5rem;     /* 24px - Section headings */
--text-3xl: 1.875rem;   /* 30px - Page headings */
--text-4xl: 2.25rem;    /* 36px - Display */
--text-5xl: 3rem;       /* 48px - Hero */
--text-6xl: 3.75rem;    /* 60px - Hero large */
--text-7xl: 4.5rem;     /* 72px - Hero XL */
```

### Line Height

```css
--leading-tight: 1.1;    /* Headings */
--leading-snug: 1.25;    /* Subheadings */
--leading-normal: 1.5;   /* Body text */
--leading-relaxed: 1.625; /* Long-form */
--leading-loose: 2;      /* Spaced text */
```

### Typography Rules

- Headings always use serif
- Code, data, timestamps always use mono
- UI chrome uses sans
- Maximum line length: 65-75 characters
- Baseline grid: 8px
- No arbitrary font sizes
- Optical sizing enabled for large text

## Color System

### OKLCH Foundation

All colors are defined in OKLCH (Oklab Lightness, Chroma, Hue) for perceptual uniformity.

```css
/* OKLCH: oklch(L% C H) */
/* L = Lightness (0-100%) */
/* C = Chroma (0-0.4) */
/* H = Hue (0-360) */
```

### Theme Generation

Light and dark themes are generated by adjusting L (lightness) only:

```css
/* Light theme */
--background: oklch(98% 0.01 0);
--foreground: oklch(15% 0.01 0);

/* Dark theme (L inverted) */
--background: oklch(12% 0.01 0);
--foreground: oklch(95% 0.01 0);
```

### Neutral Scale

```css
--neutral-50:  oklch(98% 0.005 0);
--neutral-100: oklch(95% 0.005 0);
--neutral-200: oklch(90% 0.005 0);
--neutral-300: oklch(82% 0.005 0);
--neutral-400: oklch(65% 0.01 0);
--neutral-500: oklch(50% 0.01 0);
--neutral-600: oklch(40% 0.01 0);
--neutral-700: oklch(30% 0.01 0);
--neutral-800: oklch(20% 0.01 0);
--neutral-900: oklch(15% 0.01 0);
--neutral-950: oklch(10% 0.005 0);
```

### Semantic Colors

```css
--success: oklch(65% 0.2 145);  /* Green */
--warning: oklch(75% 0.15 85);  /* Amber */
--error: oklch(60% 0.25 25);    /* Red */
--info: oklch(65% 0.15 250);    /* Blue */
```

### Contrast Requirements

| Element | Minimum Ratio | Standard |
|---------|---------------|----------|
| Body text | 4.5:1 | WCAG 2.2 AA |
| Large text (18px+) | 3:1 | WCAG 2.2 AA |
| UI components | 3:1 | WCAG 2.2 AA |
| Focus indicators | 3:1 | WCAG 2.2 AA |

### Color Rules

- No raw hex values anywhere
- All colors via CSS variables
- Dark mode via L-channel shift only
- Chroma consistent within families
- Hue consistent for brand colors

## Texture & Depth

### Generative Noise

Subtle noise overlays add tactile quality to surfaces.

```css
.surface-textured {
  background-image: url('/noise.svg');
  background-size: 200px 200px;
  opacity: 0.03;
  mix-blend-mode: overlay;
}
```

### Application Rules

| Surface | Noise | Intensity |
|---------|-------|-----------|
| Hero backgrounds | Yes | 3% |
| Card backgrounds | Yes | 2% |
| Dividers | Yes | 5% |
| Section transitions | Yes | 3% |
| Body text areas | No | — |
| Form inputs | No | — |
| Buttons | No | — |

### Performance Budget

- Noise generation: < 2ms/frame
- No JavaScript-based noise
- SVG filter preferred
- CSS `mix-blend-mode` for overlay

### Shadow Scale

```css
--shadow-sm: 0 1px 2px oklch(0% 0 0 / 0.05);
--shadow-md: 0 4px 6px oklch(0% 0 0 / 0.07);
--shadow-lg: 0 10px 15px oklch(0% 0 0 / 0.1);
--shadow-xl: 0 20px 25px oklch(0% 0 0 / 0.15);
```

### Elevation Layers

| Layer | Z-Index | Shadow | Use Case |
|-------|---------|--------|----------|
| Base | 0 | none | Page content |

| Floating | 20 | md | Dropdowns |

## Motion

### Timing Functions

```css
--ease-default: cubic-bezier(0.4, 0, 0.2, 1);
--ease-in: cubic-bezier(0.4, 0, 1, 1);
--ease-out: cubic-bezier(0, 0, 0.2, 1);
--ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

### Duration Scale

```css
--duration-instant: 50ms;
--duration-fast: 100ms;
--duration-normal: 200ms;
--duration-slow: 300ms;
--duration-slower: 500ms;
```

### Micro-Interactions

| Interaction | Duration | Easing | Property |
|-------------|----------|--------|----------|
| Button hover | 100ms | default | background, scale |
| Button press | 50ms | in | scale |
| Card hover | 200ms | out | shadow, translate |
| Input focus | 150ms | out | ring, border |
| Toggle switch | 200ms | bounce | transform |
| Menu open | 200ms | out | opacity, transform |
| Modal open | 300ms | out | opacity, scale |
| Toast enter | 300ms | out | opacity, translate |

### Motion Rules

- All animations < 200ms for interactions
- No layout shift from animation
- `prefers-reduced-motion` fully respected
- Animation communicates state change
- No decorative animation
- Spring physics for natural feel

### Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## States

### Loading States

Every component must define:
- Skeleton placeholder (pulsing animation)
- Spinner (for actions)
- Progress bar (for determinate)

### Empty States

Every list/grid must define:
- Illustration or icon (optional)
- Primary message
- Secondary description
- Call-to-action

### Error States

Every form/action must define:
- Inline field errors
- Toast notifications for system errors
- Error boundaries for crashes

## Accessibility

### Focus Management

- Visible focus indicators on all interactive elements
- Focus trap in modals and dialogs
- Skip links for navigation
- Logical tab order

### Screen Readers

- Semantic HTML
- ARIA labels for icons and controls
- Live regions for dynamic content
- Proper heading hierarchy

### Color Independence

- Never convey information by color alone
- Include icons or text for status
- Pattern differentiation for charts
