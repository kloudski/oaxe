# Launch Playbook

## Overview

This playbook defines the strategy for launching generated products with maximum viral impact. Every launch is optimized for elite dev/design Twitter and organic amplification.

## Launch Philosophy

1. **Ship with confidence** — No beta disclaimers, no "just a side project"
2. **Show, don't tell** — Screenshots and demos over descriptions
3. **Respect the audience** — No hype, no manipulation, no growth hacks
4. **Earn attention** — Quality work speaks for itself
5. **Build in public** — Share the journey, not just the destination

## Pre-Launch Checklist

### Product Readiness

- [ ] Core feature set complete
- [ ] All primary flows functional
- [ ] Loading, empty, error states implemented
- [ ] Mobile responsive
- [ ] Performance targets met (p95 < 1.2s)

### Brand Readiness

- [ ] Brand DNA finalized
- [ ] Design system complete
- [ ] Logo and wordmark ready
- [ ] Color palette documented
- [ ] Typography licensed and loaded
- [ ] Open Graph images generated
- [ ] Favicon set complete

### Launch Assets

- [ ] Landing page live
- [ ] Hero screenshot captured
- [ ] Demo video recorded (optional)
- [ ] Founder tweets drafted
- [ ] Product Hunt draft ready (optional)
- [ ] Press kit assembled (optional)

## Launch Assets Specification

### 1. Founder Tweets (3-5 Required)

**Purpose:** Primary launch announcement optimized for viral spread.

**Constraints:**
- Under 280 characters each
- No emojis
- No hashtags
- No "excited to announce"
- No "we've been working on"
- No "check it out"
- No marketing superlatives

**Tone:**
- Confident but not arrogant
- Specific about what it does
- Implies depth and quality
- Sounds like a technical founder

**Structure Options:**

**The Demo Tweet:**
```
[Product] turns [input] into [output].

Here's [specific example]:

[screenshot or video]
```

**The Technical Tweet:**
```
Built [product] to solve [problem].

[Technical detail that signals quality].

[Link]
```

**The Insight Tweet:**
```
[Observation about problem space].

So I built [product].

[What it does in one sentence].
```

**The Speed Tweet:**
```
[Time frame] from idea to shipped.

[Product] does [thing].

[Link]
```

### 2. Pinned Tweet (1 Required)

**Purpose:** Evergreen tweet that introduces the product to profile visitors.

**Structure:**
```
[Product name]: [One-line description].

[Key differentiator or insight].

[Link]

[Screenshot]
```

**Example:**
```
Atlas9: Product replication engine for builders.

Point it at any product. Get production-grade code, design, and brand.

atlas9.dev

[Hero screenshot]
```

### 3. Screenshot Spec (1 Required)

**Purpose:** The hero image that represents the product.

**Requirements:**
- 1200x675px (Twitter card) or 1920x1080px (full res)
- Shows actual product UI, not mockup
- Demonstrates primary value proposition
- High information density
- No browser chrome
- Dark mode preferred for dev audience
- Subtle noise texture for brand consistency

**Composition:**
- Hero area with product name (optional)
- Product UI taking 70%+ of frame
- Clean background (brand color or subtle gradient)
- No device frames unless mobile-specific

### 4. Hero Copy Block (1 Required)

**Purpose:** Landing page headline and subhead.

**Structure:**
```
[Headline: What it does in 3-7 words]

[Subhead: How it works or key benefit in one sentence]

[CTA Button]
```

**Constraints:**
- Headline: 3-7 words
- Subhead: 10-20 words
- No jargon
- No buzzwords
- Specific > generic

**Examples:**

```
Turn any product into code.

Point Atlas9 at Facebook, Stripe, or your own idea.
Get production-grade architecture, design, and brand.

[Start Building]
```

```
The last starter template you'll need.

Complete design system. Production architecture.
Shipping-ready from day one.

[Get Started]
```

### 5. Product Hunt Tagline (1 Required)

**Purpose:** One-line description for Product Hunt listing.

**Constraints:**
- Under 60 characters
- Clear category positioning
- Implies value immediately

**Structure:**
```
[Action verb] [what] [how/benefit]
```

**Examples:**
- "Turn any product into production-grade code"
- "Ship products with world-class design, instantly"
- "The AI that builds like a senior eng team"

## Launch Timing

### Best Times

- **Tuesday-Thursday:** Highest engagement for dev content
- **9-11 AM PT:** US coasts both active
- **Avoid:** Fridays, weekends, holidays, major tech events

### Launch Sequence

**T-7 days:**
- Finalize all assets
- Test all links
- Prepare analytics

**T-1 day:**
- Final screenshot capture
- Schedule tweets (don't auto-post, be present)
- Prepare to respond

**T-0 (Launch):**
- Post main announcement tweet
- Be present for first 2 hours
- Respond to every reply
- Retweet quality responses

**T+1 day:**
- Post follow-up technical tweet
- Share interesting early feedback
- Engage with critics constructively

**T+7 days:**
- Share learnings thread
- Post metrics if impressive
- Thank community

## Amplification Strategy

### Earned Amplification

- Quality work gets shared naturally
- Technical depth attracts senior engineers
- Visual excellence attracts designers
- Solve real problems people tweet about

### Community Engagement

- Respond to every mention
- Be helpful in replies
- Share credit with influences
- Don't ask for retweets

### No Dark Patterns

- No engagement bait
- No manufactured urgency
- No fake scarcity
- No manipulative copy
- No paid promotion (let quality speak)

## Viral Mechanics

### What Makes Dev/Design Twitter Share

1. **Visual excellence** — Looks better than expected
2. **Technical depth** — Signals expertise
3. **Solves pain** — Addresses known problem
4. **Novel approach** — Fresh perspective
5. **Open source / Free tier** — Accessible to try
6. **Speed of shipping** — Implies builder energy
7. **Polish** — Attention to detail

### Screenshot Virality Factors

- High information density
- Dark mode aesthetic
- Monospace typography visible
- Clean visual hierarchy
- Subtle animation (video)
- No placeholder content

### Copy Virality Factors

- Specificity over generality
- Technical credibility
- Implied depth
- Confidence without arrogance
- Zero marketing speak

## Activation Loop

### First-Time User Flow

1. **Land** — Clear value prop in 3 seconds
2. **See** — Demo or screenshot showing outcome
3. **Try** — Single-click to meaningful experience
4. **Succeed** — Get value in < 60 seconds
5. **Share** — Natural share moment (not forced)

### Share Triggers

- First successful generation (screenshot moment)
- Impressive output quality
- Saved time visible
- Feature discovery delight

## Metrics to Track

### Launch Day

- Tweet impressions
- Profile visits
- Landing page visits
- Sign-ups
- First actions taken

### Week One

- Daily active users
- Activation rate (signed up → value)
- Retention D1, D3, D7
- Organic mentions
- Quality of feedback

### What Success Looks Like

| Metric | Good | Great | Exceptional |
|--------|------|-------|-------------|
| Launch tweet impressions | 50K | 200K | 1M+ |
| Sign-ups day one | 500 | 2000 | 10000+ |
| D7 retention | 10% | 20% | 40%+ |
| Organic mentions week one | 50 | 200 | 1000+ |

## Post-Launch

### Week 1-2

- Respond to all feedback
- Ship quick fixes
- Collect feature requests
- Identify power users

### Week 3-4

- Ship most-requested feature
- Publish learnings post
- Engage power users
- Plan v1.1

### Month 2-3

- Establish regular shipping cadence
- Build community
- Refine positioning
- Plan v2
